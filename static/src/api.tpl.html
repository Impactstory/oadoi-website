API and dataset

There are two ways to get access to the oaDOI dataset: the API is the best bet for most uses because it's simple, fast, and easy to get running; the dataset is great for large-scale research and data mining.

Since we're dealing with 90 million articles, it takes a while to run a given version of our OA-detection algorithm across the entire dataset. This means that at any given time there are results from multiple versions of the algorithm co-existing in the database. You can generally ignore this, but for some research uses (including replicating our recent study), it's important to take into account. For any given DOI, you can check the algorithm_version key to see which algorithm generated the data we're reporting. (in the case of the study, we limited results to ones obtained from version 2 of the algorithm).

If you're using the API or dataset, we recommend you subscribe to the mailing list in order to stay up-to-date when there are changes or new features.

API

Our REST API gives anyone free, programmatic access to all of oaDOI's data. There's currently no rate limit, although if you need more than 100k calls/day you may want to download the dataset instead.

Requests must include your email, so that we can get in touch if something breaks, and so we can report usage to our funders. Add the email as a parameter at the end of the URL, like this: ?email=YOUR_EMAIL.

The API is very simple, and has just two endpoints. Both are read-only.

GET / gets information about the API. Returns a status object with version number. Try it here: https://api.oadoi.org?email=test@example.com

GET /:doi gets data about a single DOI. Try this example: https://api.oadoi.org/10.1038/nature12373?email=test@example.com

Download the dataset

The dataset behind the oaDOI API is available for download as a big CSV file. It's free for use in non-commercial academic research. Contact us if you're interested in downloading it.

If you'd like to download the dataset for commercial use, check out our service-level agreement program.

Return format

The labels for data are the same whether you're looking at JSON keys in API, or column names in the dataset. Here are the labels, and some notes describing them. Note that there may also be other experimental keys/columns at any given time, so don't count on the number of keys remaining the same, and don't count on any keys remaining the same, other than the stable ones described below:
  • algorithm_version: Number. Represents the version of the OA-detection algorithm we used to gather data on this DOI.
  • doi: the requested DOI
  • doi_resolver: String. Possible values:
    • crossref
    • datacite
  • evidence: String. A phrase summarizing the step of our OA detection process where we found the free_fulltext_url.
  • free_fulltext_url: String. The url where we found a free-to-read version of the DOI. None when no free-to-read version was found.
  • is_boai_license: Boolean. True whenever the license is cc-by, cc0, or PD. This is the highly-regarded BOAI definition of Open access
  • is_free_to_read: Boolean. True whenever the free_fulltext_url is not None.
  • is_subscription_journal: Boolean. True whenever the journal is not in the Directory of Open Access Journals or DataCite.
  • license: String. Contains the name of the Creative Commons license associated with the free_fulltext_url, whenever we find one. Example: "cc-by".
  • oa_color: String. Possible values:
    • green
    • gold
  • url: the canonical DOI URL
